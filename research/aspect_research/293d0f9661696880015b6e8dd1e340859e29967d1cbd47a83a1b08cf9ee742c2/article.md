# OpenAI — Harness engineering: leveraging Codex in an agent-first world

URL: https://openai.com/index/harness-engineering/

## Reconstructed from snippets

OpenAI's "harness engineering" essay (Feb 2026) is the canonical big-lab manifesto for the Dark Factory pattern from the OpenAI side. The team describes an internal product where "every line of code — application logic, tests, CI configuration, documentation, observability, and internal tooling — has been written by Codex," estimated to have taken about "1/10th the time it would have taken to write the code by hand."

### Core definition (quoted)

> "The term harness has emerged as a shorthand to mean everything in an AI agent except the model itself — Agent = Model + Harness."

> "Harness encodes scaffolding, feedback loops, documentation, and architectural constraints into machine-readable artifacts, which Codex agents use to execute tasks across development workflows, including code generation, testing, and observability."

### The five-month experiment

> "A three-person team started with an empty repository in late August 2025, and for five months they wrote no code themselves — every line was generated by Codex, resulting in one million lines of production code and 1,500 merged pull requests."

The team worked depth-first: "breaking down larger goals into smaller building blocks (design, code, review, test, etc), prompting the agent to construct those blocks, and using them to unlock more complex tasks. Humans interact with the system almost entirely through prompts: an engineer describes a task, runs the agent, and allows it to open a pull request."

### Key lessons (snippet quotes)

- "Give Codex a map, not a 1,000-page instruction manual. A giant instruction file crowds out the task, the code, and the relevant docs — so the agent either misses key constraints or starts optimizing for the wrong ones."
- "Harness engineering shifts human engineers' focus from implementing code to designing environments, specifying intent, and providing structured feedback."

### Why it matters for the Dark Factory thesis

This is OpenAI publicly running the Dark Factory playbook on themselves: zero human-written code, the human role compressed to spec author and harness designer. Coupled with Anthropic's 2026 Agentic Coding Trends Report, it represents both frontier labs converging on the same operating model. Coverage in InfoQ, Martin Fowler's site, and Towards AI followed within weeks.
