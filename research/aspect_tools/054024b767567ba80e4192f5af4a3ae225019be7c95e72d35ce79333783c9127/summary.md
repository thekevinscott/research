# Summary

Simon Willison's Feb 7, 2026 blog post is the canonical reference for the "Dark Factory" software development pattern. It documents StrongDM's "Software Factory" - a system where AI coding agents write and review production security code with two stated rules: code must not be written by humans, and code must not be reviewed by humans. The piece describes how three engineers (Jay Taylor, Navan Chauhan, Justin McCarthy) operate a non-interactive development pipeline driven by specs and scenarios.

The article introduces the core vocabulary now associated with dark factories: a "scenario" is an end-to-end user story stored outside the codebase (a held-out set, like in model training) that can be flexibly validated by an LLM judge; "satisfaction" is the fraction of trajectories across all scenarios that likely meet user intent. To prevent reward hacking - agents writing return true that passes tests but does nothing - StrongDM built a Digital Twin Universe of Okta, Jira, Slack, Google Docs, Drive, and Sheets, behavior-level clones run at thousands of scenarios per hour, and hide scenario descriptions from the agents under test. This post sets the conceptual frame for the rest of the dark-factory tooling ecosystem.
