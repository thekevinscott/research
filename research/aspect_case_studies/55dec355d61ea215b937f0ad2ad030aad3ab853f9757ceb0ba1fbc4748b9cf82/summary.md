# Summary

This Show HN introduces OctopusGarden, an open-source implementation of the StrongDM dark factory pattern. The author wrote and shipped a complete end-to-end "specs + scenarios + attractor + satisfaction" loop, with the explicit goal of letting anyone run a dark factory locally. Specs are markdown, scenarios are YAML and treated as a holdout set the coding agent never sees, an Attractor convergence loop drives generate -> test -> score -> feedback -> regenerate, and an LLM judge scores Satisfaction probabilistically (0-100) instead of boolean pass/fail - explicitly to prevent reward hacking.

The system also adds practitioner-grade extras: Preflight (LLM checks spec/scenario quality before any generation), Wonder/Reflect (two-phase stall recovery via high-temp diagnosis then low-temp surgical fixes), Model Escalation, Gene Transfusion (lifting patterns from exemplar codebases), and Stratified Validation. Sample specs in the repo demonstrate REST APIs, an authenticated todo app and a terminal UI - the author claims it "works with mostly standard CRUD/REST API apps." It is the most concrete "run-it-yourself" dark factory artifact in the public discourse.
