## What this is

A practitioner blog post by a working web developer who has been running Claude Code as a scheduled cloud agent on his own projects. He frames the team-level implications and pairs it with adjacent posts ("From Tasks to Swarms: Agent Teams in Claude Code", "Spec-Driven Development with Claude Code in Action") that document his hands-on workflow.

## Direct quotes / near-quotes from snippets

> "In a software factory, developers stop writing code by hand, and AI coding agents implement features and fix bugs while developers design and improve the factory."

> "Claude Code runs as a scheduled agent in the cloud, reading the codebase, writing code, running tests, and opening PRs."

> "Handoffs are the bottleneck, with the BA handing off to the PO, the PO to the dev, and the dev to QA, each handoff adding waiting time and context loss."

> "In a software factory, each team member is a product builder, where the UX designer can write a spec and hand it to an agent, and the business analyst can describe a fix and kick off an agent."

> "We already have everything we need to build software factories, teams will change, and the only variable is speed."

## Key first-hand observations

- Claude Code runs unattended in cloud and opens PRs, removing the typical handoff chain (BA -> PO -> dev -> QA).
- The role boundary blurs: anyone able to write a clear spec can drive a build, including UX designers and business analysts.
- The bottleneck moves from coding capacity to spec quality and orchestration design - matching what Ona and Nate's Newsletter found.
