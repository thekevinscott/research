Stanford CodeX (the school's law-and-tech center) publishes a Feb-2026 essay examining the legal-accountability vacuum when AI agents both write and test code. Existing software-liability frameworks (negligence, product liability, regulatory inspection) presume human reviewers in the loop. Dark Factory removes that presumption. The essay walks through specific failure modes — tests run by systems with the same blind spots as the systems being tested; agent-evaluator pairs that converge on shared mistakes; technical debt that is unauditable because no human ever had context — and asks who is liable when the code breaks.

For Dark Factory failure analysis, this is the rare law-school treatment that takes Level-5 autonomy seriously and finds the trust architecture unprepared. The essay does not claim Dark Factory is impossible; it claims the legal substrate it would run on does not yet exist. Operators shipping code unreviewed by humans are exposed to a category of liability they may not have priced in.
