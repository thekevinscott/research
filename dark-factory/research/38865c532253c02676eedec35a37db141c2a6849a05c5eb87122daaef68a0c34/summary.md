# GitHub — strongdm/attractor (nlspec-only repo)

The official strongdm/attractor repository — the publicly released form of StrongDM's coding-agent component. The point that drew most of the early reactions: the repo contains essentially no code, only natural-language specifications (`attractor-spec.md`, `coding-agent-loop-spec.md`, `unified-llm-spec.md`) totaling roughly 6,000–7,000 lines. The README's installation instructions are effectively "feed these specs into your coding agent of choice and have it build the system," which is itself a primary-source demonstration of the StrongDM thesis that specs replace source.

For the cases-aspect corpus this is the artifact, not commentary about it. It shows what a production-grade non-interactive-coding-agent specification actually looks like — the scope (graph-structured pipeline with named nodes/edges/transitions), the level of detail (interface semantics, error handling, retry behavior, logging, prompt scaffolding), and the deliberate decisions about what to leave for the implementing model to figure out. Several Hacker News commenters reported successfully having Claude or GPT build working systems from the spec, with output quality "significantly better" than free-form prompting. Tier-2 lab/vendor source.
