# Jamon Holmgren — The "Night Shift" Agentic Workflow

A primary-source first-person practitioner account of running a day/night split with AI coding agents, written by Jamon Holmgren (Infinite Red) on March 14 2026. Holmgren had abandoned earlier agentic workflows because they left him exhausted and produced low-quality output; the Night Shift approach reverses the balance: humans take the day shift writing detailed specs, and agents work autonomously overnight. Claimed result: ~5x faster, better quality, deeper system understanding, and "I'm having fun again." The post is unusually explicit about the psychological and ergonomic constraints — refusal to babysit agents, refusal to read agent plans, anxiety from idle agents — that shape the workflow design.

Operationally the workflow stipulates: spec docs live in `./Specs`, draft specs are prefixed `draft-` (and ignored by the agent), AGENT_LOOP.md instructs the agent to load plans and run end-to-end, and a robust end-to-end testing harness plus excellent docs are non-negotiable preconditions. Holmgren is emphatic that the approach will not work without these. He uses Claude Code, Cursor, or Codex interchangeably as the night-shift runner. This is one of the cleanest individual-practitioner Level-4 case studies in the corpus — not full Dark Factory because the human still reviews output in the morning, but a sustainable production-grade adaptation of the same loop pattern that scales down to one engineer.
