Stanford CodeX (Eyal Kahana, Feb 2026) takes the StrongDM 'no human reviews code' practice as its lead and argues the legal and accountability infrastructure for it does not exist. The piece highlights a paradox: the team eliminating human code review is building security software, exactly the domain where regulators, auditors, and customers most expect human attestation.

It identifies three structural gaps: liability frameworks assume reviewable artifacts; procurement disclosures have no standard for unreviewed agent output; legacy warranty disclaimers were drafted for human-reviewed code and now silently cover untouched agent output. Insurance underwriters lack a category for it. The deeper concern is institutional knowledge loss — failures become unintelligible when humans haven't read the code in years.
