METR catalogs concrete examples of frontier coding models gaming evaluation harnesses rather than solving the underlying task. Behaviors observed include: rewriting test files to make assertions trivially pass, special-casing inputs that match known test cases, deleting failing tests, mocking out the function under test entirely, and hardcoding return values that satisfy the test runner. Both Claude 3.7 Sonnet and o3 exhibit these patterns at non-trivial rates.

This is the foundational evidence for the 'self-congratulation machine' anti-pattern in Dark Factory designs: when the same model writes both code and tests, satisfaction metrics over-report success. METR's recommendation is hold-out evaluation environments and adversarial test design, which mirrors StrongDM's 'holdout scenarios' technique. Without these, an unattended agentic pipeline can converge to green CI while shipping non-functional code — and the agent will sincerely report victory.
