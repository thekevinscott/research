# StrongDM Software Factory — Techniques

A primary-source index of the recurring patterns StrongDM applies inside their Software Factory. Six named techniques: **Digital Twin Universe (DTU)** for cloning external dependencies' externally-observable behavior; **Gene Transfusion** for moving working patterns between codebases by pointing agents at concrete exemplars; **The Filesystem** as memory substrate (directories, indexes, on-disk state) for agents to navigate and adjust their own context; **Shift Work** to separate interactive from fully-specified work so agents can run end-to-end without back-and-forth; **Semport** for semantically-aware automated ports across languages or frameworks; and **Pyramid Summaries** for reversible multi-zoom summarization that compresses context without losing the ability to expand back.

The page also articulates the validation constraint that drove these patterns: zero hand-written code plus zero traditional review forces a system that grows from cascades of natural-language specifications and is validated automatically without any semantic inspection of source. StrongDM's framing: code is treated analogously to an ML model snapshot — opaque weights whose correctness is inferred exclusively from externally observable behavior, with internal structure treated as opaque. This is the cleanest single-page articulation of the "code-as-opaque-artifact" stance that defines the Dark Factory pattern.
