# Calvin French-Owen (calv.info) — Coding Agents in Feb 2026

A primary-source practitioner snapshot from Calvin French-Owen, who helped launch OpenAI's Codex web product and uses Claude Code, Codex, and Cursor extensively. The post lays out his concrete day-to-day workflow choosing between agents based on time available and how long he wants the agent to run autonomously — Opus for planning and orchestrating his terminal because it handles context windows efficiently and uses sub-agents (Explore via Haiku) effectively, Codex for raw correctness because it produces fewer bugs in his experience. He pays for Claude Max, ChatGPT Pro, and Cursor Pro+ simultaneously and considers it among the best money he spends.

The piece's broader contribution is a practitioner articulation of context discipline as the core skill of running coding agents at Levels 4–5. French-Owen lays out concrete corollaries: chunk work to fit context windows, treat compaction as lossy, externalize state into the filesystem (plan docs with stage checkboxes), stay in the "smart half" of the context window (cites Dex Horthy's "dumb zone"), and assume the agent doesn't know what it doesn't know. He references OpenAI's "harness engineering" post on structuring many markdown files for progressive disclosure. Notable as a senior-practitioner Level-4-trending-toward-5 case study by someone with insider OpenAI/Codex experience, written for an audience already running these agents in production.
