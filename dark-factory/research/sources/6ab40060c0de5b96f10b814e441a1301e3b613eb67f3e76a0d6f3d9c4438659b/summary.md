# GitHub — foundatron/octopusgarden (A Dark Software Factory)

The OctopusGarden README — the most complete open-source implementation of the StrongDM-style Dark Factory pattern publicly available as runnable code (rather than spec-only like Attractor). The README documents the pipeline end-to-end: a `spec.md` plus a `scenarios.md` go in, an orchestrator dispatches generation to a coding agent, the result is built in an ephemeral Docker sandbox, scenarios run against the live build, an LLM-as-judge produces probabilistic satisfaction scores, and failures feed back as targeted prompts until the build converges or a budget cap is hit.

For the cases-aspect corpus the README is valuable as a hands-on artifact: it shows how the "scenarios as holdout set" idea looks when you have to actually run it (scenarios live outside the agent's reachable filesystem, are never shown verbatim, and only failure summaries are returned to the generator), what convergence criteria practitioners pick (typical defaults of 90%+ scenario satisfaction across N runs), and what guardrails matter (Docker isolation, time/cost budgets, max-iteration caps). It's a working reference implementation that other small teams can copy, fork, or measure their own factory against.
