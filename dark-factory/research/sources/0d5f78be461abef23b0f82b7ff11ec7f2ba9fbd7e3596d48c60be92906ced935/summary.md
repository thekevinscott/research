Namanyay Goel's 'Karpathy's Vibe Coding Movement Considered Harmful' argues that vibe coding's core failure mode is moving the human contributor from the relatively easy task of writing code (where you build mental model as you go) to the much harder task of debugging code you've never read. When something goes wrong — and at vibe-coding scale, something always goes wrong — the human has no scaffolding to diagnose it. The post cites instances of vibe-coded codebases that had to be wholly rewritten the moment a non-trivial bug surfaced, because no one understood the code well enough to repair it.

For Dark Factory failure analysis, this is the 'mean time to recover' argument: even if mean time to ship goes down, mean time to recover from incidents goes up because no human has context. In a fully unattended Level-5 setup, the recovery problem compounds: you cannot ask the agent that wrote the code to fix it, because the agent's session is gone and the new agent has no more context than you do. Goel's argument is widely cited in skeptic discourse.
