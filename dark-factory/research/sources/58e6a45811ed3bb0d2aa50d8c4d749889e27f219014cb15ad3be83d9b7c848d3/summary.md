Stripe's engineering blog introduces 'Minions' — fully unattended one-shot coding agents that take a ticket or spec and produce a complete pull request. Every Minion runs in an isolated dev VM identical to a human engineer's, spinning up in ~10 seconds with the Ruby payments codebase pre-warmed. Tools are heavily curated (~15 of 400+ internal tools) to avoid token paralysis, and retry budget is capped at two attempts to avoid wasting compute.

Stripe explicitly retains human review at the merge stage, so this is Dark-Factory-on-write but not on-read. The throughput claim — 1000+ PRs/week — is the largest published number for unattended agent code generation in fintech/payments, and the architecture description doubles as the most concrete public template for an internal-tooling Dark Factory on a giant Ruby monorepo.
