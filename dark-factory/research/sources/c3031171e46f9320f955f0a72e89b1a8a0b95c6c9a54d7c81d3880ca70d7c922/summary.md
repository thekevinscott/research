Aikido Security researcher Charlie Eriksen registered react-codeshift on npm — a name no one had ever published, but one that LLMs repeatedly hallucinated in code suggestions. Within months, 237 GitHub repositories had auto-installed it via npx (often invoked by AI 'agent skills' for framework migrations). Had the package been malicious, it would have executed in 237 developers' environments without review. ~20% of LLM-recommended packages don't exist on npm or PyPI, so the attack surface is enormous.

For Dark Factory analysis, slopsquatting is the supply-chain failure mode that scales with autonomy: in a Level-5 setup, every hallucinated dependency is a silent install. USENIX Security 2025 quantifies the issue (16 models tested, 576k samples; 38% of hallucinations are conflations, 13% typos, 51% pure fabrications). Mitigation requires sandbox-installation, allowlisted registries, or SBOM-based provenance — none of which appear in the canonical Dark Factory blueprints.
