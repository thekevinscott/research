Trend Micro's threat-intel writeup describes slopsquatting as a supply-chain attack class purpose-built for AI coding workflows. Attackers monitor LLM hallucinations across npm, PyPI, and similar registries; when a plausible non-existent package name is repeatedly suggested, attackers register it themselves and ship malware. Trend Micro cites Lasso Security's huggingface-cli case (>30,000 downloads of a malicious-but-trusted-because-AI-recommended package) and estimates 440,000+ potentially poisoned package names already exist in the ecosystem.

For Dark Factory failure analysis, this is the failure mode where the cost is borne by victims, not the agent operator: a Dark Factory agent installs a malicious package, the malware runs, and downstream users are compromised — but the agent's metrics show 'task completed'. Dark Factory designs that don't sandbox `npm install` / `pip install` / `cargo add` are externalizing security cost while collecting throughput credit.
