# StrongDM Software Factory — Attractor

Primary-source documentation for **Attractor**, StrongDM's non-interactive coding-agent component. Attractor is structured as a graph of nodes forming a generative SDLC: nodes correspond to phases of work governed by a core prompt (Implement, Identify, Optimize, Validate), and edges between nodes are expressed in natural language and evaluated by the LLM itself. Execution traverses the graph until convergence or a termination condition. The framing is deliberate: Attractor is not a chatbot wrapper — it's a directed pipeline that an agent walks deterministically while the LLM judges each transition.

The page also serves as a bridge to community implementations and the open-source release. The Attractor GitHub repo contains specs only (no runtime code) and instructs users to feed them to a coding agent of choice. The product page links to several community ports/forks — including Foundatron's OctopusGarden — and emphasizes the design intent of operating end-to-end once the work is fully specified, distinguishing "non-interactive" coding agents from interactive assistants like Copilot or Cursor. Together with the principles and techniques pages, this is the canonical statement of how Attractor sits inside the Software Factory loop.
