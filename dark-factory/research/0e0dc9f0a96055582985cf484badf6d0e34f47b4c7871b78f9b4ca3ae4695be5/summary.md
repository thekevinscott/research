# InfoQ — Stripe Engineers Deploy Minions, Autonomous Agents Producing Thousands of Pull Requests Weekly

InfoQ's news write-up of Stripe's Minions release (March 2026) — a synthesis-tier secondary source covering the Stripe primary blog posts (which are JS-rendered and harder to capture verbatim). The piece states the headline numbers: Minions now produce 1,300+ PRs/week (up from 1,000 in earlier trials), all changes are human-reviewed but contain no human-written code, and the code Minions manage supports more than $1 trillion in annual payment volume across complex regulatory and financial-institution dependencies. Minions are described as one-shot end-to-end agents — different from interactive Copilot/Cursor — invoked from Slack threads, bug reports, or feature requests.

The article's contribution beyond reporting numbers is technical context: Minions evolved from an internal fork of **Goose**, the open-source coding agent originally developed by Block, customized for Stripe's LLM infrastructure. They are orchestrated using "blueprints" — workflows defined in code that mix deterministic routines and flexible agent loops, described as "agent skills interwoven with code" for a balance of efficiency and adaptability. Reliability comes from CI/CD pipelines, automated tests, and static analysis gating before human review. Engineers note Minions perform best on well-defined tasks (config adjustments, dependency upgrades, minor refactoring). Important nuance: Stripe is a Level-4 case (human review still in the loop), not strict Dark Factory, but the production-volume metrics are the most consequential cited in 2026.
